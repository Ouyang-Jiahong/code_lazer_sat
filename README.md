# 空间目标探测任务规划与评估任务书

---

## 一、实验目的

1. **建立数学模型**：能够针对空间目标探测任务场景，合理选择决策变量，分析约束条件和目标函数，并构建数学模型。
2. **设计任务规划算法**：基于所建模型，设计合适的数值求解方法（如线性规划、启发式算法等），并使用 Python 或 MATLAB 实现探测方案的生成。
3. **构建评估体系**：结合课堂讲授内容或自行查阅文献，构建探测方案的评估指标体系，设计评估方法，并通过仿真验证探测方案的效能。

---

## 二、实验内容

### 1. 问题建模

- **任务场景描述**：
  - 使用一批相控阵雷达和机械跟踪雷达对给定的空间目标进行探测。
  - 雷达参数（部署位置、探测能力）详见 `sensorData.xlsx`。
  - 目标轨道信息（TLE）详见 `objectData.txt`。
  - 每颗目标的探测需求（测站数量、探测次数、最短探测时间等）详见 `requireData.xlsx`。
  - 探测时间段为：**2021年10月14日 4时0分0秒 至 2021年10月15日 4时0分0秒（UTCG时间）**。

- **建模要求**：
  - 分析任务中的关键决策变量（如雷达与目标的探测分配关系、时间调度等）。
  - 明确约束条件（如雷达探测容量、探测时间限制、可见弧段限制等）。
  - 构建目标函数（如最大化探测覆盖率、最小化资源消耗、提高重点目标优先级等）。
  - 给出完整的数学表达式，包括变量定义、目标函数、约束条件。

---

### 2. 方案生成

- **算法设计要求**：
  - 根据所建立的探测任务规划模型，选择合适的数值优化方法进行求解。
    - 可选方法包括但不限于：混合整数线性规划（MILP）、遗传算法（GA）、模拟退火（SA）、贪心策略等。
  - 编写程序实现探测计划的生成：
    - 输入：雷达数据、目标轨道数据、探测需求、可见弧段数据（`usableArcs.mat`, `simDate.mat`）。
    - 输出：每部雷达在指定时间段内对每个目标的探测安排（起止时间、是否探测、探测顺序等）。
  - 要求代码结构清晰、注释完整、输入输出明确。

---

### 3. 方案评估

- **评估体系构建要求**：
  - 设计合理的评估指标体系，涵盖以下方面：
    - 探测覆盖率（成功探测的目标比例）
    - 雷达资源利用率（实际使用的雷达资源占总可用资源的比例）
    - 重点目标完成率（高优先级目标的探测完成情况）
    - 探测时间均衡性（不同雷达之间的探测负载分布）
    - 弧段完整性（是否满足探测弧段的连续性要求）

- **评估方法实现要求**：
  - 编写仿真程序，根据生成的探测方案计算上述各项评估指标。
  - 输出评估结果表格或可视化图表（如柱状图、雷达图等）。
  - 对比不同探测方案的性能差异，分析优劣。

---

## 三、实验成果要求

### 1. 仿真程序

- **语言**：Python 或 MATLAB
- **功能要求**：
  - 完整实现探测任务规划模型的求解过程。
  - 支持读取 `sensorData.xlsx`、`objectData.txt`、`requireData.xlsx` 和 `usableArcs.mat` 数据。
  - 输出探测计划文件（可为 `.txt`、`.csv` 或 `.xlsx` 格式）。
  - 实现探测方案评估模块，输出评估结果。
- **代码规范**：
  - 注释清晰、结构合理、命名规范。
  - 主函数入口明确，便于运行测试。

### 2. 实验报告

- **格式要求**：按模板撰写，排版整洁、图文并茂。
- **内容要求**：
  1. **问题建模**：
     - 决策变量定义
     - 约束条件详细说明
     - 目标函数设计及数学表达
  2. **探测任务规划方法说明**：
     - 所采用的算法原理简介
     - 算法流程图或伪代码
     - 参数设置说明
  3. **探测任务方案生成结果**：
     - 展示部分探测计划结果（建议以表格或甘特图形式展示）
     - 说明方案中各雷达的任务分配情况
  4. **探测任务方案评估**：
     - 各评估指标的设计与计算方法
     - 评估结果展示（表格/图表）
     - 结果分析与结论
  5. **拓展思考**：
     - 结合前期“近距离观测轨迹规划”等实验，谈对航天任务规划的理解。
     - 对任务评估方法的认识与改进设想。

## 五、附录：数据格式说明

### 1. `simDate.mat`

- 6×1441 矩阵，每一列为 `[年, 月, 日, 时, 分, 秒]` 时间戳。

### 2. `usableArcs.mat`

- 结构体数据，字段如下：

| 字段名        | 描述 |
|---------------|------|
| `SatID`       | 卫星编号 |
| `RadarID`     | 雷达编号 |
| `ArcChain`    | N×2 矩阵，表示可见弧段的开始列索引和结束列索引（对应 `simDate` 的列号） |
| `ArcDuration` | N×1 矩阵，表示每个可见弧段的持续时间（单位：秒） |

### 3. `requireData.xlsx`

- 每一行代表一颗空间目标的探测需求，字段如下：

| 行号 | 参数名称（单位）         |
|------|--------------------------|
| 1    | 目标编号                 |
| 2    | 半长轴（km）             |
| 3    | 偏心率                   |
| 4    | 轨道倾角 (°)             |
| 5    | 升交点赤经 (°)           |
| 6    | 近地点幅角 (°)           |
| 7    | 平近点角 (°)             |
| 8    | 优先级（数值越大越重要） |
| 9    | 需要的测站数量           |
| 10   | 需要的观测时间 (min)     |
| 11   | 需要的弧段数量           |
